Catchment Delineation API

A REST API for automated river catchment delineation using 30 m SRTM elevation data. Given a bounding box or polygon, it downloads the DEM, computes D8 flow routing, extracts stream networks, traces upstream catchment boundaries, and produces GeoJSON results, an HTML report, and a ZIP export.

Base URL: All endpoints are relative to your deployment root, e.g. https://your-app.railway.app/api/v1/…

Methodology

  • DEM source: SRTM GL1 (~30 m) via OpenTopography API. Falls back to AWS Terrain Tiles if rate-limited.
  • Flow routing: D8 algorithm — each cell drains to one of 8 neighbours. Implemented via pysheds with pit-filling and depression-filling.
  • Stream extraction: Cells where flow accumulation ≥ acc_threshold are classified as streams.
  • Catchment delineation: BFS upstream from the snapped outlet, clipped to the original study polygon.
  • Strahler ordering: Applied at segment level to the river network graph.

Sessions & Authentication

There is no API key required. The API is open access.

All state (loaded DEM, results, rasters) is scoped to a session. Create a session first, then pass the session_id to every subsequent call in one of three ways:

MethodExample
JSON body field"session_id": "abc123"
Query parameter?session_id=abc123
Request headerX-Session-Id: abc123
Memory: Sessions are stored in-memory on the server. Always call DELETE /api/v1/session/<id> when done to free resources. Sessions are not persisted across server restarts.

Quickstart

Complete workflow from scratch to downloadable results:

1 — Create a session

curl -s -X POST https://your-app.railway.app/api/v1/session \
  -H "Content-Type: application/json"
# → {"session_id": "a3f9b2...", "created_at": "2025-01-01T00:00:00Z"}
SID="a3f9b2..."

2 — Fetch DEM (by bounding box)

curl -s -X POST https://your-app.railway.app/api/v1/dem/fetch \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SID\",\"bbox\":{\"south\":51.80,\"north\":51.88,\"west\":-3.55,\"east\":-3.40}}"
# → {"bounds":{...}, "res_m":30.9, "default_acc":150, "n_river_segments":582, ...}
# Store default_acc — use it as acc_threshold in /delineate

3 — Delineate a catchment

curl -s -X POST https://your-app.railway.app/api/v1/delineate \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SID\",\"lat\":51.83,\"lon\":-3.47,\"acc_threshold\":150}"
# → {"catchment":{GeoJSON}, "outlet":{GeoJSON}, "stats":{"area_km2":34.2,...}}

4 — Download intermediate rasters

# List available rasters
curl -s "https://your-app.railway.app/api/v1/rasters/list?session_id=$SID"

# Download DEM GeoTIFF
curl -s "https://your-app.railway.app/api/v1/rasters/download/dem?session_id=$SID" -o dem.tif

# Download flow direction
curl -s "https://your-app.railway.app/api/v1/rasters/download/flow_direction?session_id=$SID" -o fdir.tif

# Download flow accumulation
curl -s "https://your-app.railway.app/api/v1/rasters/download/flow_accumulation?session_id=$SID" -o acc.tif

5 — Generate HTML report

curl -s -X POST https://your-app.railway.app/api/v1/report \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SID\",\"title\":\"My Catchment\",\"author\":\"Agent\"}" \
  | python -c "import sys,json; open('report.html','w').write(json.load(sys.stdin)['html'])"

6 — Export ZIP (map PNG + GeoJSON)

TOKEN=$(curl -s -X POST https://your-app.railway.app/api/v1/export \
  -H "Content-Type: application/json" \
  -d "{\"session_id\":\"$SID\",\"title\":\"My Catchment\"}" \
  | python -c "import sys,json; print(json.load(sys.stdin)['token'])")

curl -s "https://your-app.railway.app/api/v1/export/download/$TOKEN" -o results.zip

7 — Cleanup

curl -s -X DELETE "https://your-app.railway.app/api/v1/session/$SID"
# → {"ok": true}

Error Format

All errors return an appropriate HTTP status code with a JSON body. No HTML error pages. Structural state-loss errors also include a machine-readable code field so clients can branch without parsing English prose.

{"error": "No DEM loaded. Call /api/v1/dem/fetch first.", "code": "NO_DEM"}

// 500 errors also include a traceback:
{"error": "...", "trace": "Traceback (most recent call last):\n..."}

HTTP status

StatusMeaning
400Missing or invalid parameters / precondition not met
404Unknown session, raster_id, or token
500Server-side processing error (includes trace)

Structured code values

CodeMeaning
SESSION_INVALIDMissing, unknown, or expired session_id. Create a new one via POST /api/v1/session.
NO_DEMThe session has no loaded DEM. Call POST /api/v1/dem/fetch first.
NO_RESULTSNo catchments delineated yet. Call POST /api/v1/delineate before report / export.
OUT_OF_BOUNDSThe supplied lat/lon is outside the loaded DEM bounds.
CATCHMENT_TOO_SMALLDelineation produced fewer than 10 cells — lower acc_threshold or move upstream.
VECTORISE_FAILEDInternal: couldn't build a polygon from the catchment mask.

Create Session

POST /api/v1/session

Create a new isolated analysis session. Must be called before any other endpoint. Returns a session_id to pass to all subsequent requests.

Request body

Empty — no fields required.

Response

FieldTypeDescription
session_idstringUnique session identifier — pass to all subsequent calls
created_atstringISO-8601 UTC creation timestamp
curl -X POST https://your-app.railway.app/api/v1/session
{"session_id": "a3f9b2c1d4e5f6...", "created_at": "2025-01-01T12:00:00Z"}

Delete Session

DELETE /api/v1/session/<session_id>

Free a session and all its associated data from server memory — DEM arrays, computed rasters, and delineation results. Always call this when you're done to avoid memory leaks.

curl -X DELETE https://your-app.railway.app/api/v1/session/a3f9b2c1d4e5f6
{"ok": true}

DEM Fetch

POST /api/v1/dem/fetch

Download and process a DEM for a study area. This is the first step after creating a session. It downloads 30 m SRTM data, conditions it (pit-filling, depression-filling, flat resolution), computes D8 flow direction and accumulation, extracts the river network, and caches all intermediate rasters in memory.

Synchronous & slow: This call blocks for 30–120 seconds depending on area size. For areas larger than ~1° × 1°, expect longer times.

Request body

FieldTypeRequiredDescription
session_idstringrequiredSession identifier
bboxobjectone of{south, north, west, east} in decimal degrees WGS84
polygonobjectone ofGeoJSON Polygon geometry (type + coordinates)
sourcestringoptional "auto" (default) tries OpenTopography SRTM first, falls back to AWS Terrain Tiles on failure. "srtm" forces SRTM only — returns HTTP 500 if OpenTopography is unreachable (useful when you need the 30 m resolution guaranteed). "aws" skips SRTM and fetches AWS Terrain Tiles directly (~76 m, no API key) — handy when the OpenTopography key is rate-limited or for very large areas. Omitting the field preserves the original behaviour, so existing integrations need no change.

Response

FieldTypeDescription
boundsobjectActual DEM extent {south,north,west,east}
res_mnumberDEM cell size in metres
shapearray[rows, cols]
acc_p95number95th-percentile flow accumulation value
default_accintegerUse as acc_threshold in /delineate and /rivers
n_river_segmentsintegerNumber of river segments at default threshold
dem_sourcestringData source label
# Using bbox
curl -X POST https://your-app.railway.app/api/v1/dem/fetch \
  -H "Content-Type: application/json" \
  -d '{
    "session_id": "a3f9b2...",
    "bbox": {"south": 51.80, "north": 51.88, "west": -3.55, "east": -3.40}
  }'
{
  "bounds":           {"south": 51.80, "north": 51.88, "west": -3.55, "east": -3.40},
  "res_m":            30.9,
  "shape":            [288, 540],
  "acc_p95":          130.0,
  "default_acc":      150,
  "n_river_segments": 582,
  "dem_source":       "OpenTopography SRTM GL1 (30 m)"
}

Rasters Download

GET /api/v1/rasters/download/<raster_id>?session_id=…

Stream a georeferenced GeoTIFF (LZW-compressed, WGS84/EPSG:4326). Rasters are pre-built at /dem/fetch time — downloads are instant, no recomputation.

Tip: Open directly in QGIS, rasterio, or GDAL. The affine transform and CRS are embedded in the file.
raster_iddtypenodataDescription
demfloat32-9999Elevation in metres
flow_directionint160D8 direction codes (see reference)
flow_accumulationfloat32-1Upstream cell count
# Download all three rasters
for RID in dem flow_direction flow_accumulation; do
  curl -s "https://your-app.railway.app/api/v1/rasters/download/$RID?session_id=a3f9b2..." \
    -o "${RID}.tif"
done

# Verify with rasterio (Python)
import rasterio
with rasterio.open("dem.tif") as ds:
    print(ds.crs, ds.dtypes, ds.bounds)

Export ZIP

POST /api/v1/export

Build a ZIP containing a print-quality map PNG and all GeoJSON layers. Returns a single-use download token — GET the download_url once to retrieve the file.

Single-use token: The download token is consumed on first use. Call /api/v1/export again to get a new one.

ZIP contents

  • catchment_map.png — print-quality map with basemap, hillshade, rivers, and outlets
  • catchments.geojson — catchment boundary polygons
  • outlets.geojson — snapped outlet points
  • rivers.geojson — full river network

Request body

FieldTypeRequiredDescription
session_idstringrequired
titlestringoptionalMap title and ZIP filename stem
acc_thresholdintegeroptionalRiver threshold for map
paperstringoptional"A4" | "A3" | "Letter" (default "A4")
orientationstringoptional"landscape" | "portrait" (default "landscape")
outletsarrayoptional[{id, name, color}]

Response

FieldTypeDescription
tokenstringSingle-use download token
filenamestringSuggested ZIP filename
download_urlstringFull URL — GET this once to download

Download ZIP

GET /api/v1/export/download/<token>

Download the pre-built export ZIP using the token returned by /api/v1/export. The token is consumed on first use — returns 404 if already used or expired.

D8 Flow Direction Codes

The flow_direction raster encodes each cell's drainage direction as an integer. The encoding matches the standard ArcGIS / TauDEM D8 convention:

ValueDirectionDescription
64NNorth (up)
128NENortheast
1EEast (right)
2SESoutheast
4SSouth (down)
8SWSouthwest
16WWest (left)
32NWNorthwest
0—NoData / outside study area

Raster Types Reference

IDdtypenodataUnitNotes
demfloat32-9999metresElevation above sea level. NaN cells (outside polygon) written as -9999.
flow_directionint160D8 codeSee D8 table above. Cells outside polygon are 0.
flow_accumulationfloat32-1cellsNumber of upstream DEM cells. High values = main channels. Use as stream threshold.

All rasters share the same affine transform, CRS (WGS84 / EPSG:4326), and grid dimensions. They can be overlaid directly in QGIS or processed together with rasterio / GDAL.
