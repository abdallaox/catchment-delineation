1 · DEM Acquisition

SRTM GL1 (30 m) tiles are downloaded from the OpenTopography API and clipped to the drawn polygon. If the API key is rate-limited (50 calls / 24 h), the tool falls back automatically to AWS Terrain Tiles (~76 m/px, no API key required). The raster is cropped with rasterio.mask(crop=True) so only cells inside the polygon are retained.

2 · DEM Conditioning — Walled DEM

NODATA cells outside the polygon boundary are replaced with max_elev + 5000 m before flow routing. This "wall" prevents pysheds from routing flow through masked areas and keeps all analysis strictly inside the drawn polygon. After routing, fdir and acc are zeroed outside the valid mask as a second safeguard.

3 · Flow Routing (D8)

The conditioned DEM is processed with pysheds using the D8 (eight-direction) method:

  • Pit filling (fill_pits)
  • Depression filling (fill_depressions)
  • Flat resolution (resolve_flats)
  • Flow direction (flowdir) — 8 cardinal/diagonal directions encoded as 1, 2, 4, 8, 16, 32, 64, 128
  • Flow accumulation (accumulation) — number of upstream contributing cells
4 · River Network & Strahler Ordering

Stream cells are identified where acc ≥ threshold (user-adjustable). River segments are traced by walking the D8 flow graph between topological nodes — headwaters (n_upstream = 0), confluences (n_upstream ≥ 2), and outlets. Strahler ordering is then assigned at the segment level:

  • Headwater segments are always order 1 (regardless of cell-level accumulation artefacts upstream)
  • At a confluence, if two or more tributaries share the same maximum order N, the downstream segment becomes order N+1; otherwise it inherits order N
  • Segments are processed in topological order (ascending accumulation) to guarantee consistent propagation

This segment-level approach eliminates the cell-level noise that arises from applying Strahler rules per pixel. River polylines are smoothed with Chaikin corner-cutting (3 iterations) for display.

5 · Outlet Snapping

A click on the map snaps to the nearest stream cell (minimum Euclidean distance) within a 15-cell search radius. This preserves the user's intended location rather than jumping to the highest-accumulation cell in the area.

6 · Catchment Delineation

Starting from the snapped outlet cell, a BFS (breadth-first search) traversal walks upstream through the D8 flow-direction grid. The resulting boolean mask is ANDed with the polygon valid-mask to guarantee no catchment cells leak outside the drawn boundary. The mask is then vectorised to a polygon with rasterio.features.shapes.

7 · Upload External Layers

Any vector dataset can be overlaid on the map by uploading a Shapefile ZIP (containing .shp, .dbf, .shx, .prj), a bare .shp, or a GeoJSON file. The server reprojects non-WGS84 datasets to EPSG:4326 with GeoPandas before returning GeoJSON to the browser. Each uploaded layer receives a unique colour and can be removed individually from the sidebar.

8 · Export

Results are packaged as a ZIP archive containing:

  • catchment_results.gpkg (or Shapefiles) — outlet points and river network (vector, WGS84)
  • dem.tif — clipped SRTM elevation raster (Float32, LZW-compressed)
  • flow_accumulation.tif — upstream contributing-cell count raster (Float32)
  • flow_direction.tif — D8 direction codes raster (Int16; values: 1, 2, 4, 8, 16, 32, 64, 128)

All rasters preserve the original WGS84 projection and affine transform of the clipped DEM. Catchment polygons are not included in the export — delineation boundaries are derived on-the-fly from the flow-direction raster and are always reproducible from it.
